# Labelpilot.de — UI Design System for German B2B Label Infrastructure

## 1. Purpose

This document defines the visual design system for **Labelpilot.de**.

Labelpilot.de must not look like:

```txt
cheap online print shop
generic AI landing page
colorful sticker website
unfinished Tailwind template
```

Labelpilot.de must look like:

```txt
modern German B2B SaaS
industrial label commerce
trusted procurement platform
technical but simple
premium but not luxury
conversion-focused
```

The design goal is not decoration.

The design goal is trust, clarity and conversion.

---

## 2. Strategic Verdict

## 1. Hüküm: Yap

The current Phase 1 site should be redesigned using this design system before adding more features.

## 2. Neden: Kısa ve matematiksel

If the site looks weak:

```txt
German buyer trust drops
quote conversion drops
sample box conversion drops
Google Ads CAC rises
brand positioning weakens
```

A better UI can improve:

```txt
lead conversion
trust
time-on-page
form completion
B2B credibility
```

Design is not vanity. It is conversion infrastructure.

## 6. Color System

## 6.1 Primary Palette

Use a restrained palette.

Recommended:

```txt
Primary Navy: #0B1220
Deep Slate: #111827
Text Dark: #172033
Muted Text: #64748B
Border: #E2E8F0
Soft Background: #F8FAFC
White: #FFFFFF
```

## 6.2 Accent Colors

Use one primary accent.

Recommended accent:

```txt
Trust Blue: #2563EB
Blue Hover: #1D4ED8
Soft Blue BG: #EFF6FF
```

Optional success/accent:

```txt
Process Green: #16A34A
Soft Green BG: #F0FDF4
```

Warning:

```txt
Amber: #F59E0B
Soft Amber BG: #FFFBEB
```

Error:

```txt
Red: #DC2626
Soft Red BG: #FEF2F2
```

## 6.3 Color Rules

Do:

```txt
use white background heavily
use navy for text/headers
use blue for primary CTA
use green only for process/trust states
use subtle grey borders
```

Do not:

```txt
use orange everywhere
use rainbow product cards
use heavy gradients
use black background on every section
use low contrast text
```

---

## 7. Typography

## 7.1 Font Direction

Use modern readable sans-serif.

Recommended:

```txt
Inter
Geist
system sans-serif fallback
```

Do not use decorative fonts.

## 7.2 Type Scale

Recommended:

```txt
Hero H1 desktop: 56–72px
Hero H1 mobile: 36–44px
Page H1: 44–56px
Section H2: 30–40px
H3: 22–28px
Body large: 18–20px
Body: 16–18px
Small: 14px
Micro: 12px
```

## 7.3 Typography Rules

Do:

```txt
use strong line-height
keep line length readable
use high contrast
use German compound words carefully
use bold only for emphasis
```

Do not:

```txt
make paragraphs too wide
center-align long text
use all caps heavily
use tiny grey text for important information
```

## 28. Final Verdict

## 28. Design Benchmark Addendum

Study these product-page patterns:

- Labelprint24 for B2B trust and structured label detail pages
- Flyeralarm for dense but scannable procurement-style product organization
- Onlineprinters for live-price feel and serious B2B table structure
- Sticker Mule / StickerApp / Vistaprint for CTA clarity and streamlined configurator flow
- Avery / Vistaprint for format and template education

Do not copy:

- broad generic catalog sprawl
- playful sticker-shop tone
- Turkish-exporter / generic matbaa visual language

Our visual rule remains:

```txt
German B2B SaaS
navy/slate
Trust Blue CTA
white/grey surfaces
calculator cards
saved-design / reorder moat
```

Strategic design line:

```txt
Competitors hide exact prices behind configurators.
Labelpilot.de should show transparent indicative tiers and exact package inclusions directly on the page.
```

The design must communicate:

```txt
This company remembers my labels.
This is safer than emailing files to a printer.
This is for serious product brands.
This is easy to reorder.
```

If the site only communicates:

```txt
we print labels
```

the design failed.
