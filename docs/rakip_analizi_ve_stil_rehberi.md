# Rakip Analizi ve Stil Rehberi — Labelpilot.de

> Amaç: Labelpilot.de’yi standart AI/SaaS şablonu hissinden çıkarıp, premium Alman B2B rulo etiket üreticisi + tekrar sipariş altyapısı olarak konumlandırmak.
>
> Kural: Rakiplerden **tasarım disiplini** alınır; layout, metin, renk, CTA veya görsel yapı **kopyalanmaz**.

---

## 0. Stratejik Hüküm

### 1. Hüküm: Yap

Labelpilot.de tasarımı; “ucuz online matbaa”, “oyuncak sticker sitesi” veya “AI ile yapılmış SaaS landing page” gibi görünmemeli.

Doğru konum:

```txt
Premium Alman B2B rulo etiket üreticisi + kayıtlı tasarım + hızlı tekrar sipariş altyapısı.
```

### 2. Neden: Kısa ve matematiksel

Pazardaki büyük oyuncuların gücü:

```txt
Flyeralarm / WIRmachenDRUCK / Onlineprinters = trafik + katalog + fiyat + güven
Labelprint24 / etikett.de = teknik etiket uzmanlığı + üretim güveni
Sticker Mule / StickerApp = basit checkout + net CTA + görsel sadelik
Avery / Vistaprint = template + küçük işletme dili
```

Labelpilot’in kazanabileceği boşluk:

```txt
ilk sipariş = müşteri kazanımı
kayıtlı baskı dosyası = kilitlenme
30 saniyelik tekrar sipariş = LTV artışı
lot/SKT değişken veri = supplement müşterisi için gerçek operasyonel değer
```

### 3. Riskler + Çözüm

| Risk | Sonuç | Çözüm |
|---|---|---|
| Site AI/SaaS gibi görünürse | Alman B2B alıcı güvenmez | Gerçek ürün görselleri, teknik tablo, kısa net metin |
| Sadece “etiket basıyoruz” denirse | Büyük matbaalarla fiyat savaşına girilir | “Gespeicherte Druckdaten + Nachbestellung” ana fark yapılır |
| Çok fazla metin yazılırsa | Premium algı düşer | Görsel ağırlıklı, kısa teknik bloklar |
| Çok fazla ürün eklenirse | Matbaa pazarı bataklığına girilir | MVP: opak PP, transparan PP, Musterbox, Nachbestellung |

### 4. Daha iyi alternatif

Genel online matbaa görünümü yerine şu mimari kullanılmalı:

```txt
Hero: ürün + tekrar sipariş vaadi
Material: opak / transparan PP karşılaştırma
Packages: 1.000 / 2.000 / 5.000 / 10.000 net fiyat kartları
Workflow: Druckdaten → Proof → Produktion → Nachbestellung
Portal Moat: Gespeicherte Designs + Versionen + Lot/SKT
Trust: Musterbox + technische Prüfung + Versand nach Deutschland
```

### 5. Beklenen çıktı

```txt
Daha yüksek güven
Daha düşük bounce
Daha net B2B algısı
Daha güçlü quote request
Daha savunulabilir fiyat
Daha yüksek tekrar sipariş ihtimali
```

### 6. Fırsat maliyeti

AI şablonu gibi devam ederse:

```txt
Flyeralarm seni ezer.
WIRmachenDRUCK fiyatla ezer.
Labelprint24 teknik güvenle ezer.
Sticker Mule checkout sadeliğiyle ezer.
Vistaprint template tarafında ezer.
```

Labelpilot’in tek yüksek-ROI yolu:

```txt
premium ürün etiketi + kayıtlı dosya + hızlı tekrar sipariş + B2B operasyon netliği.
```

# 1. Benzersiz Değer Teklifi & Marka Tonu (Copywriting)

## 1.1 Klişe Temizliği

Aşağıdaki AI kokan kalıplar kesinlikle yasak:

```txt
Yenilikçi çözümler sunuyoruz.
Geleceğin etiket deneyimini inşa ediyoruz.
Markanızı bir üst seviyeye taşıyoruz.
İhtiyaçlarınıza özel çözümler.
Kaliteden ödün vermeden.
Modern ve güvenilir hizmet.
Uçtan uca çözüm ortağınız.
Dijital dönüşümünüz için.
Sektör lideri çözümler.
Profesyonel ekibimizle yanınızdayız.
Premium kalite, uygun fiyat.
Hızlı, kaliteli, güvenilir.
```

Bu ifadeler para kazandırmaz. Herkes yazıyor. Hiçbir şey kanıtlamıyor.

## 1.2 Labelpilot’in Ana Değer Cümlesi

### Kötü

```txt
Wir bieten innovative Etikettenlösungen für moderne Marken.
```

### İyi

```txt
PP-Rollenetiketten für Lebensmittel-, Getränke- und Supplement-Marken — mit gespeicherten Druckdaten für schnellere Nachbestellungen.
```

Neden iyi?

```txt
Ürün net: PP-Rollenetiketten
Hedef net: Lebensmittel, Getränke, Supplement
Fark net: gespeicherte Druckdaten
Sonuç net: schnellere Nachbestellungen
```

## 1.3 Ses Tonu (Tone of Voice)

Labelpilot’in sesi şu olmalı:

```txt
kendinden emin
sade
teknik ama boğmayan
Alman B2B ciddiyetinde
insan eliyle yazılmış gibi kısa
abartısız

```

## 1.4 Metin Kuralları

### Kural 1 — Daha az kelime, daha net mesaj

Kötü:

```txt
Mit unseren innovativen und maßgeschneiderten Etikettenlösungen helfen wir modernen Marken, ihre Produkte optimal am Markt zu positionieren.
```

İyi:

```txt
Bedruckte PP-Rollenetiketten für wiederkehrende Produktlinien.
```

### Kural 2 — Her başlık bir karar verdirmeli

Kötü başlık:

```txt
Unsere Lösungen
```

İyi başlık:

```txt
Einmal freigeben. Später schneller nachbestellen.
```

### Kural 3 — Teknik iddia tabloyla desteklenmeli

Metin tek başına satmaz. Teknik iddia varsa yanında tablo olmalı.

Örnek:

```txt
Material: Opake PP-Folie / Transparente PP-Folie
Format: 100 × 200 mm
Mindestmenge: 1.000 Stück
Empfohlen: 5.000 Stück
Druck: CMYK Digitaldruck
Proof: vor Produktion
Nachbestellung: mit gespeicherter Druckdatei
```

### Kural 4 — Alman müşteriye üretim yeri saklanmaz

Kötü:

```txt
Deutsche Qualität zum besten Preis.
```

İyi:

```txt
Für Deutschland geplant, kosteneffizient in der Türkei produziert, mit klarer Druckdatenprüfung und Versand nach Deutschland.
```

### Kural 5 — Hukuki uygunluk iddiası yapılmaz

Kötü:

```txt
Rechtssichere Lebensmitteletiketten.
```

İyi:

```txt
Wir prüfen Druckdaten technisch. Rechtliche Pflichtangaben, Zutaten, Nährwerte und Health Claims bleiben in Verantwortung des Kunden.
```

## 1.5 Sayfa Başlığı Stili

Başlıklar kısa, ağır ve gerçek olmalı.

### Homepage H1 alternatifleri

```txt
PP-Rollenetiketten für Marken, die nachbestellen.
```

```txt
Produktetiketten auf Rolle — gespeichert für die nächste Bestellung.
```

```txt
Etiketten für Lebensmittel-, Getränke- und Supplement-Marken.
```

En güçlü kombinasyon:


```txt
Labelpilot produziert bedruckte PP-Etiketten auf Rolle und speichert freigegebene Druckdaten, damit wiederkehrende Bestellungen schneller und sicherer laufen.
```

## 2.1 Tasarım Hükmü

### Yap

## 2.2 Font Çiftleri

Aşağıdaki font çiftleri uygulanabilir. Ama font lisansı kontrol edilmeden production’a alınmaz.

## Seçenek A — En Dengeli Premium B2B

```txt
Başlık: Söhne / Suisse Int’l / Neue Haas Grotesk alternatifi
Gövde: IBM Plex Sans / Source Sans 3
Kullanım: Alman B2B, teknik, güvenilir, modern
```

Ücretsiz uygulanabilir alternatif:

```txt

Gövde: IBM Plex Sans
```

Karakter:

```txt
net
premium
teknik
fazla moda değil
uzun ömürlü
```

## Seçenek B — Daha Lüks ve Editoryal

```txt
Başlık: Fraunces / Cormorant Garamond / Playfair Display
Gövde: Manrope / IBM Plex Sans
Kullanım: premium, butik, yüksek marjlı ürün algısı
```


```txt

Gövde: DM Sans / Geist / Manrope
Kullanım: teknoloji + üretim + portal hissi
```


# 3. Yapısal Asimetri ve Layout Mantığı

## 3.1 Temel Layout Hükmü

### Yap

Organik, editorial, ürün odaklı, asimetrik ama kontrollü layout.

### Yapma

```txt
3 kolon eşit kart
6 özellik kartı grid
her bölümde aynı max-width + aynı card radius
hero sol metin sağ dashboard mockup
her kartta icon + başlık + 2 satır açıklama
```

Bu yapı AI şablon kokar.

## 3.2 Sayfa Ritim Sistemi

Sayfa ritmi şu sırayı izlemeli:

```txt
1. Büyük net hero
2. Ürün/malzeme görseli
3. Kısa teknik gerçek
4. Fiyat / paket karar alanı
5. İş akışı
6. Reorder farkı
7. Musterbox / güven
8. Sektör kullanımları
9. FAQ
10. Final CTA
```

Her bölüm aynı yoğunlukta olmamalı. Premium sayfa ritmi dalgalanmalı.

```txt
Büyük sessiz alan → yoğun teknik tablo → görsel nefes → fiyat kararı → kısa proof akışı
```

## 3.3 Asimetrik Hero Modeli

Standart 50/50 layout yerine:

```txt
Desktop:
Sol: 7 kolon metin + CTA
Sağ: 5 kolon ürün görseli
Görsel: hafif aşağı taşmalı, kutuya hapsolmayacak
Alt: trust strip tam genişlik ama düşük kontrast
```

Örnek yapı:

```txt
[H1 büyük, 2 satır]
[Alt metin, maksimum 2 satır]
[CTA primary] [CTA secondary]
[3 mikro güven sinyali]

                 [etiket rulosu + materyal close-up]
                 [küçük floating spec label]
```

Spec label örneği:

```txt
100 × 200 mm
PP opak / transparent
Proof vor Produktion
```

## 3.4 Kart Grid Yerine “Editorial Modules”

Yasak standart:

```txt
[card] [card] [card]
[card] [card] [card]
```

Kullanılacak yapı:

```txt
Büyük ürün modülü: 60%
İki küçük teknik modül: 40%
Altına yatay proof/reorder şeridi
```

Örnek:

```txt
┌──────────────────────────────┬──────────────┐
│ Büyük material comparison    │ Mini spec    │
│ opak vs transparent          ├──────────────┤
│                              │ Proof note   │
└──────────────────────────────┴──────────────┘
```

## 3.5 Ürün Paketleri Layout’u

Fiyat kartları eşit olmamalı. 5.000 adet Growth paketi görsel olarak öne çıkmalı.

```txt
1.000 Starter       normal küçük
2.000 Reorder Ready normal
5.000 Growth        daha yüksek, daha koyu, “Empfohlen”
10.000 Pro          teknik, güçlü ama Growth kadar parlak değil
20.000+ Angebot     yatay B2B quote bloğu
```

Growth kartı:

```txt
scale: 1.03
shadow daha derin
başlık daha güçlü
unit price belirgin
CTA primary
```

Diğer kartlar:

```txt
daha sakin
border yerine soft shadow
CTA secondary
```

## 3.6 “Organik Boşluk” Kuralları

Premium boşluk, boş alan değil; güven sinyalidir.

Tailwind önerileri:

```txt
Hero section: py-28 md:py-36 lg:py-40
Normal section: py-24 md:py-32
Yoğun teknik tablo bölümü: py-20 md:py-24
Final CTA: py-28 md:py-36
Container: max-w-[1180px] veya max-w-[1240px]
Metin kolon genişliği: max-w-[680px]
Okuma paragrafları: max-w-[620px]
```

Yasak:

```txt
Her section py-16.
Her alan aynı max-width.
Hero’da sıkışık metin.
Tablo ve kartların birbirine yapışması.
```

## 3.7 Bölüm Geçişleri

AI tasarımda bölümler kutu kutu ayrılır. Labelpilot’te geçişler daha sakin olmalı.

Kullanılacak yöntemler:

```txt
zemin rengi çok hafif değişsin: #FFFDF8 → #F7F2E8
ince top border yerine geniş boşluk
görsel objeler section sınırını hafif taşsın
bölüm başlıkları her zaman üstte değil, bazen yan kolon içinde olsun
```

## 3.8 Teknik Tablolar

Rakiplerin teknik güveni tabloyla verdiği görülüyor. Labelpilot de bunu kullanmalı.

Tablo stili:

```txt
arka plan: transparent veya warm white

```

Yasak:

## 3.9 Reorder Modülü Layout’u

Bu bölüm sitenin moat alanı. Dashboard gibi değil, üretim hafızası gibi görünmeli.

Önerilen yapı:

```txt
Sol: büyük başlık
Einmal freigeben. Später schneller nachbestellen.

Sağ: saved design card
- Produkt: Supplement Dose
- Material: Transparente PP-Folie
- Format: 100 × 200 mm
- Version: v3 freigegeben
- Letzte Bestellung: 5.000 Stück
```

Altında 3 adım:

```txt
Druckdaten gespeichert
Version freigegeben
Menge wählen und nachbestellen
```

Ama bu 3 adım standart icon-card grid olmamalı. İnce zaman çizgisi gibi yapılmalı.

## 3.10 Mobil Layout

Mobilde asimetri korunmalı ama karmaşa olmamalı.

```txt
Hero görseli metnin altına iner.
Fiyat kartları yatay scroll değil, tek tek stacked olur.
Growth kartı ilk ekranda açıkça öne çıkar.

```

Mobil sticky CTA:

```txt

```

# 4. Mikro Etkileşimler ve Derinlik

## 4.1 Derinlik Hükmü

### Yap

Sessiz, pahalı, fiziksel ürün hissi veren derinlik.

### Yapma

```txt
kalın gri border
her kartta aynı shadow
hover’da zıplayan kart
gradient glow
neon mavi focus dışı
```

## 4.2 Soft Shadow Kütüphanesi

Border yerine düşük opaklıklı sıcak gölgeler kullanılmalı.

```css
--shadow-paper: 0 1px 2px rgba(17, 16, 14, 0.04), 0 8px 24px rgba(17, 16, 14, 0.06);
--shadow-product: 0 18px 55px rgba(17, 16, 14, 0.10);
--shadow-float: 0 24px 80px rgba(17, 16, 14, 0.14);
--shadow-cta: 0 10px 28px rgba(17, 16, 14, 0.18);
--shadow-brass: 0 18px 45px rgba(176, 138, 69, 0.18);
```

Tailwind karşılığı:

```txt
shadow-[0_1px_2px_rgba(17,16,14,0.04),0_8px_24px_rgba(17,16,14,0.06)]
shadow-[0_18px_55px_rgba(17,16,14,0.10)]
shadow-[0_24px_80px_rgba(17,16,14,0.14)]
```

## 4.3 Radius Sistemi

Her şeyi `rounded-2xl` yapmak AI kokar.

```txt
Küçük badge: rounded-full
Button: rounded-full veya rounded-[14px]
Fiyat kartı: rounded-[28px]
Büyük ürün görsel paneli: rounded-[36px]
Teknik tablo: radius yok veya rounded-[18px]
Input: rounded-[12px]
```

## 4.4 Border Kullanımı

Border tamamen yasak değil; ama ana derinlik aracı olmamalı.

Kullanılacak border:

```txt
1px solid #E5DED2
sadece tablo satırı, input, ayrım çizgisi, teknik alan
```

Yasak:

```txt
her kartta border-gray-200
border + shadow + bg-gray-50 üçlüsü
```

## 4.5 Hover Geçişleri

Hover hareketleri pahalı saat gibi olmalı: sessiz, yavaş, kontrollü.

```txt
duration-500
ease-out
translate-y-[-2px] maksimum
scale sadece görselde 1.015
opacity geçişi 0.85 → 1
```

Button hover:

```css
transition: transform 500ms ease, background-color 500ms ease, box-shadow 500ms ease;
```

Tailwind:

```txt
transition-all duration-500 ease-out hover:-translate-y-0.5
```
